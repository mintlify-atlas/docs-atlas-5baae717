## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed on your machine
- A terminal

## Run with Docker

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/dotnetfactory/fluid-calendar.git
    cd fluid-calendar
    ```
  </Step>

  <Step title="Configure your environment">
    Copy the example environment file:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and set these required values:

    ```bash .env
    DATABASE_URL=postgresql://fluid:fluid@db:5432/fluid_calendar
    NEXTAUTH_URL=http://localhost:3000
    NEXTAUTH_SECRET=your-random-secret-key-min-32-chars
    ```

    <Tip>
      Generate a secure secret with: `openssl rand -base64 32`
    </Tip>
  </Step>

  <Step title="Start the application">
    ```bash
    docker compose up -d
    ```

    Docker will pull the pre-built image and start both FluidCalendar and PostgreSQL automatically.
  </Step>

  <Step title="Open the app">
    Visit [http://localhost:3000](http://localhost:3000) in your browser.

    On first launch, you'll be guided through the initial setup to create your admin account.
  </Step>
</Steps>

## First steps after setup

Once you're logged in, here's what to do next:

<Steps>
  <Step title="Connect a calendar">
    Go to **Settings → Integrations** and connect your Google Calendar or Outlook account. This lets FluidCalendar see your existing events when scheduling tasks.

    - [Connect Google Calendar →](/integrations/google-calendar)
    - [Connect Outlook Calendar →](/integrations/outlook-calendar)
  </Step>

  <Step title="Configure work hours">
    Go to **Settings → Auto-Schedule** and set your working days and hours. FluidCalendar will only schedule tasks during these times.
  </Step>

  <Step title="Create your first task">
    Click the **Tasks** tab and add a task with an estimated duration. Enable **Auto-schedule** on the task, then click **Auto-schedule all** to let FluidCalendar find the best time for it.
  </Step>

  <Step title="Check your calendar">
    Switch to the **Calendar** view to see your auto-scheduled tasks placed on your calendar alongside your existing events.
  </Step>
</Steps>

## Useful Docker commands

```bash
# View application logs
docker compose logs -f

# Stop the application
docker compose down

# Update to the latest version
docker pull eibrahim/fluid-calendar:latest
docker compose up -d

# Reset the database (caution: deletes all data)
docker compose down -v
docker compose up -d

# Access the database CLI
docker compose exec db psql -U fluid -d fluid_calendar
```
